// INSTALLATION

In 4 Schritten startklar

01
Herunterladen
Lade die passende Version für dein Betriebssystem oben im Download-Bereich herunter. Für die portable Version (.exe / .app) ist keine Installation nötig.
02
Starten
Doppelklick auf die heruntergeladene Datei. Beim ersten Start erscheint eventuell ein Windows-Sicherheitshinweis — auf „Trotzdem ausführen" klicken.
03
Anmelden
Gib deinen WebUntis-Schulnamen, Benutzernamen und Passwort ein. Optional: „Angemeldet bleiben" aktivieren für automatisches Login.
04
Fertig!
Die App lädt automatisch alle Daten und cached 20 Wochen Stundenplan im Hintergrund. Ab jetzt hast du auch offline Zugriff auf deinen Plan.
Aus dem Quellcode starten
Falls du Python bereits installiert hast, kannst du WebUntis Desktop direkt aus dem Quellcode starten:
# 1. Abhängigkeiten installieren
pip install PyQt6 webuntis requests

# 2. App starten
python main.py

// FAQ

Häufige Fragen

Ja. Wenn du „Angemeldet bleiben" aktivierst, werden deine Zugangsdaten lokal auf deinem Gerät verschlüsselt (Base64-kodiert) gespeichert. Es werden keine Daten an externe Server übertragen — alle Verbindungen gehen ausschließlich an deinen WebUntis-Schulserver.
Der KI-Assistent nutzt Meta LLaMA 3.1 8B via OpenRouter. Für die Nutzung benötigst du einen kostenlosen API-Key von openrouter.ai. Das verwendete Modell ist kostenlos. Den Key trägst du einmalig in der App oder direkt in der Konfigurationsdatei ein.
Ja, teilweise. Die App cached beim Start automatisch 20 Wochen Stundenplan lokal auf deinem Gerät. Damit hast du auch ohne aktive Internetverbindung Zugriff auf deinen Stundenplan. Live-Daten wie Vertretungen oder neue Nachrichten benötigen jedoch eine Verbindung.
Ja! WebUntis Desktop hat eingebaute Proxy-Unterstützung. In der Datei main.py kannst du PROXY_ENABLED = True setzen und deine Proxy-Adresse und den Port eintragen. Die App testet die Verbindung beim Start automatisch.
WebUntis Desktop ist in Python geschrieben und nutzt PyQt6 für die grafische Oberfläche sowie die Python-Bibliothek webuntis für die API-Anbindung. Die App ist mit Nuitka oder PyInstaller zu einer einzelnen ausführbaren Datei kompiliert.
